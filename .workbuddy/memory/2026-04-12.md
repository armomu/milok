# 2026-04-12

## 首页改造 - 分页/刷新/搜索升级

改造首页 Tab 列表，对接本地接口 `http://192.168.1.38:8085/station`，响应格式为 `{code, message, data: {data:[...], total}}`。

### 变更内容

- **ApiService.kt**：添加 `ApiResponse<T>` 和 `PagedData<T>` 包装类，`getStations` 返回类型改为 `Response<ApiResponse<PagedData<StationDto>>>`
- **Repositories.kt**：添加 `PagedResult<T>` 分页结果类，`StationRepository.getStations` 返回 `Resource<PagedResult<Station>>`
- **StationRepositoryImpl.kt**：解析新响应格式，返回 `PagedResult`
- **HomeViewModel.kt**：
  - 移除 `@HiltViewModel`/`@Inject`，改用 `MutableState` 状态管理（`var uiState by mutableStateOf(...)`）
  - 新增 `isRefreshing`、`isLoadingMore`、`totalCount`、`hasMore` 状态字段
  - 新增 `refresh()` 下拉刷新、`loadMore()` 加载更多方法
  - 添加 `ViewModelProvider.Factory` 工厂方法
- **HomeViewModelEntryPoint.kt**（新建）：Hilt EntryPoint 接口，供 HomeScreen 获取 `StationRepository`
- **HomeScreen.kt**：
  - 通过 `EntryPointAccessors` + `viewModel(factory = ...)` 创建 HomeViewModel（不用 hiltViewModel）
  - 使用 Material3 `PullToRefreshBox` 实现下拉刷新
  - 监听 `LazyListState` 滚动到底部触发 `loadMore()`
  - 底部显示加载中 indicator 和"已加载全部 N 条数据"提示

## 修复首页列表顶部/底部多余空白

- **原因**：`MainScreen`（外层 Scaffold）把 `paddingValues`（含底部导航栏高度）通过 `modifier` 传给 `HomeScreen`；`HomeScreen`（内层 Scaffold）在 content 里又用 `modifier.padding(paddingValues)` 把自己的 TopAppBar padding 叠加上去，导致顶部/底部各多出一段空白。
- **修复**：
  1. `HomeScreen` 的 Scaffold 加上 `modifier = modifier`，将外层传入的 padding 作用在 Scaffold 根容器上（规避底部导航栏）
  2. `content` 内部的 Column 改为 `Modifier.fillMaxSize().padding(paddingValues)`，只消费内层 Scaffold 自己的 TopAppBar 高度，不再叠加外层 modifier

- 错误：`CLEARTEXT communication to 192.168.1.38 not permitted by network security policy`
- 新建 `app/src/main/res/xml/network_security_config.xml`，配置 `<domain-config cleartextTrafficPermitted="true">` 允许对 `192.168.1.38` 的 HTTP 请求
- 在 `AndroidManifest.xml` 的 `<application>` 中添加 `android:networkSecurityConfig="@xml/network_security_config"`
